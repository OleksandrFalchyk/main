# РОЗДІЛ 7. Модель даних

Модель даних описує основні сутності системи та зв’язки між ними. Вона відображає структуру бази даних інформаційної системи обліку студентів.

## Основні сутності

### 1. Студент
- id
- прізвище
- ім’я
- дата народження
- номер залікової книжки
- група_id
- статус

### 2. Група
- id
- назва групи
- спеціальність
- курс

### 3. Викладач
- id
- ПІБ
- кафедра

### 4. Дисципліна
- id
- назва
- викладач_id

### 5. Оцінка
- id
- студент_id
- дисципліна_id
- бал
- дата виставлення

### 6. Користувач
- id
- логін
- пароль
- роль

---

## Зв’язки між сутностями

- Один студент належить до однієї групи.
- Одна група може містити багато студентів.
- Один викладач може викладати багато дисциплін.
- Один студент може мати багато оцінок.
- Кожна оцінка пов’язана з конкретним студентом та дисципліною.
- Користувач пов’язаний із певною роллю (адміністратор, викладач, студент, працівник деканату).

---

## Спрощене текстове представлення ER-моделі

Група (1) -------- (N) Студент  
Студент (1) -------- (N) Оцінка  
Дисципліна (1) -------- (N) Оцінка  
Викладач (1) -------- (N) Дисципліна  

Модель даних забезпечує цілісність інформації та дозволяє ефективно організувати зберігання даних у базі.
