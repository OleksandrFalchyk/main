# РОЗДІЛ 4. Функціональні сценарії (User Stories)

User Stories описують функціональність системи з точки зору користувача та допомагають зрозуміти, які можливості повинна забезпечувати система.

---

## User Stories для адміністратора

- Як адміністратор, я хочу створювати облікові записи користувачів, щоб забезпечити доступ до системи.
- Як адміністратор, я хочу призначати ролі користувачам, щоб обмежити їх права доступу.
- Як адміністратор, я хочу видаляти або блокувати користувачів, щоб забезпечити безпеку системи.

---

## User Stories для працівника деканату

- Як працівник деканату, я хочу додавати нових студентів у систему, щоб вести їх облік.
- Як працівник деканату, я хочу редагувати дані студентів, щоб підтримувати інформацію актуальною.
- Як працівник деканату, я хочу формувати списки студентів за групами, щоб контролювати навчальний процес.
- Як працівник деканату, я хочу переглядати статистику успішності, щоб аналізувати результати навчання.

---

## User Stories для викладача

- Як викладач, я хочу переглядати список студентів своєї групи, щоб вести облік відвідування.
- Як викладач, я хочу виставляти оцінки студентам, щоб фіксувати результати навчання.
- Як викладач, я хочу редагувати оцінки, щоб виправляти можливі помилки.

---

## User Stories для студента

- Як студент, я хочу переглядати свої оцінки, щоб контролювати свою успішність.
- Як студент, я хочу переглядати інформацію про свою групу, щоб бути поінформованим про навчальний процес.
