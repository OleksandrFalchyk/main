# РОЗДІЛ 5. Відображення User Stories у UML-нотації

Для відображення функціональних можливостей системи використано UML-діаграму варіантів використання (Use Case Diagram).

## Актори системи:
- Адміністратор
- Працівник деканату
- Викладач
- Студент

## Варіанти використання (Use Cases):

### Адміністратор:
- Створення користувача
- Призначення ролі
- Блокування користувача

### Працівник деканату:
- Додавання студента
- Редагування даних студента
- Формування списків груп
- Перегляд статистики успішності

### Викладач:
- Перегляд списку групи
- Виставлення оцінок
- Редагування оцінок

### Студент:
- Перегляд власних оцінок
- Перегляд інформації про групу

---

## Структурне представлення Use Case діаграми

                 +----------------------------------+
                 |  ІС обліку студентів             |
                 +----------------------------------+
        Адміністратор ----> (Створення користувача)
        Адміністратор ----> (Призначення ролі)
        Адміністратор ----> (Блокування користувача)

        Працівник деканату ----> (Додавання студента)
        Працівник деканату ----> (Редагування даних)
        Працівник деканату ----> (Формування груп)

        Викладач ----> (Виставлення оцінок)
        Викладач ----> (Перегляд списку групи)

        Студент ----> (Перегляд власних оцінок)
