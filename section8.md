# РОЗДІЛ 8. Обґрунтування вибору типу бази даних

Для інформаційної системи обліку студентів доцільно використовувати реляційну базу даних. Такий тип бази даних добре підходить для систем, у яких інформація має чітку структуру та існують зв’язки між різними сутностями.

У даній системі зберігаються дані про студентів, групи, дисципліни, викладачів та оцінки. Між цими об’єктами існують логічні зв’язки: студент належить до певної групи, студент отримує оцінки з дисциплін, а дисципліни можуть бути пов’язані з викладачами. Реляційна база даних дозволяє зручно організувати такі зв’язки за допомогою таблиць і ключів.

Ще однією перевагою реляційних баз даних є забезпечення цілісності даних. Використання первинних і зовнішніх ключів дозволяє уникнути помилок та дублювання інформації. Також реляційні бази даних підтримують мову запитів SQL, що дозволяє легко отримувати потрібну інформацію, формувати звіти та виконувати пошук.

Крім того, реляційні системи керування базами даних, такі як MySQL або PostgreSQL, є надійними, широко використовуються та добре підходять для навчальних і інформаційних систем.

Отже, використання реляційної бази даних є оптимальним рішенням для реалізації інформаційної системи обліку студентів.
